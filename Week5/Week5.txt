1. stat 구조체의 멤버

stat 구조체는 파일의 메타데이터를 저장하는 구조체다.
stat 구조체는 다음과 같은 주요 멤버들이 포함되어 있다.

st_dev: 파일이 위치한 장치의 ID(장치 번호).
st_ino: 파일의 inode 번호(파일의 고유 식별자).
st_mode: 파일의 종류와 접근 권한을 나타냄.
st_nlink: 파일의 하드 링크 수(참조 횟수).
st_uid: 파일 소유자의 사용자 ID.
st_gid: 파일 소유자의 그룹 ID.
st_rdev: 특수 파일일 경우, 장치 ID.
st_size: 파일 크기(바이트 단위).
st_blksize: 파일 시스템 I/O의 기본 블록 크기.
st_blocks: 파일에 할당된 블록 수.
st_atime: 파일에 마지막으로 접근한 시간.
st_mtime: 파일이 마지막으로 수정된 시간.
st_ctime: inode가 마지막으로 변경된 시간.

2. stat 구조체 중 st_mode 

st_mode는 파일의 종류와 접근 권한을 나타내는 필드다. 
이 값은 여러 비트 값을 통해 파일의 특정 속성을 나타낸다.

파일 종류:

S_IFREG: 일반 파일.
S_IFDIR: 디렉토리.
S_IFLNK: 심볼릭 링크.
S_IFCHR: 문자 장치 파일.
S_IFBLK: 블록 장치 파일.
S_IFIFO: FIFO(파이프).
S_IFSOCK: 소켓 파일.

파일 접근 권한:

파일 권한은 사용자, 그룹, 기타 사용자로 나뉘며, 
각각의 권한은 읽기(4), 쓰기(2), 실행(1)으로 구성된다.

S_IRUSR: 소유자에게 읽기 권한.
S_IWUSR: 소유자에게 쓰기 권한.
S_IXUSR: 소유자에게 실행 권한.
S_IRGRP: 그룹에 읽기 권한.
S_IWGRP: 그룹에 쓰기 권한.
S_IXGRP: 그룹에 실행 권한.
S_IROTH: 기타 사용자에게 읽기 권한.
S_IWOTH: 기타 사용자에게 쓰기 권한.
S_IXOTH: 기타 사용자에게 실행 권한.

3. 하드 링크와 심볼릭 링크

하드 링크 (Hard Link):

같은 파일 시스템 내에서 하나의 파일에 여러 개의 이름을 부여할 수 있게 해준다.
하드 링크는 원본 파일과 동일한 inode를 가리키며, 동일한 데이터 블록을 참조한다.
원본 파일을 삭제해도 다른 하드 링크를 통해 파일에 접근이 가능하다.
단, 디렉토리에 대한 하드 링크는 만들 수 없다

심볼릭 링크 (Symbolic Link):

심볼릭 링크는 원본 파일의 경로를 가리키는 별도의 파일이다.
다른 파일 시스템에서도 생성할 수 있으며, 디렉토리에 대한 링크도 가능하다.
원본 파일이 삭제되면 심볼릭 링크는 사용할 수 없다.
심볼릭 링크 자체는 독립적인 inode와 데이터 블록을 가진다.


이러한 특성 때문에 하드 링크는 실제 파일을 복사하지 않고 여러 이름을 부여하는 데 유용하고 심볼릭 링크는 경로를 참조하므로 파일 위치나 이름이 바뀌어도 유연하게 관리할 수 있다는 유용하다.
